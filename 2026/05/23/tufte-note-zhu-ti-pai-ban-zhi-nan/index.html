        tufte-note 主题排版指南：边注、宽图与长文阅读

        Contents

        1. 普通 Markdown 仍然可用
        2. 用 newthought 开启段落

        tufte-note 是我给这个 Hexo 博客启用的一套长文阅读主题。它基于 Tufte CSS，最明显的特点是正文比较克制，右侧留出边注区，适合放解释、引用、图例和一些不想打断正文的补充信息。

        这篇文章记录一下在 tufte-note 主题下写博客时常用的排版方法。普通 Markdown 当然可以直接写，但如果想用出 Tufte 风格，就需要在 Markdown 里穿插少量 HTML。

        当前主题：站点根目录的 _config.yml 中配置了 theme: tufte-note。主题文件位于 themes/tufte-note，其中 vendored 的 Tufte CSS 版本是 1.8.0。

        普通 Markdown 仍然可用

        最基本的标题、段落、列表、引用、代码块、表格、图片都不需要特殊处理。也就是说，原来的文章迁移到 tufte-note 后，一般不用改正文内容。

        $$

        用 marginnote 写无编号边注

        marginnote 是最常用的边注。它没有编号，适合放背景解释、术语提醒、图片说明、参考链接等信息。

        正文继续向下写。桌面端会把边注放到右侧空白区域；移动端空间不够时，主题会把边注折叠到正文流里，避免挤压阅读区域。

        写作建议：边注应该服务正文，而不是变成另一个正文。若一段话离开边注后正文就看不懂，说明它应该回到正文里。

        我更推荐把 marginnote 放在相关段落的开头或段落之前，这样右侧边注的位置更容易和正文内容对齐。

        用 sidenote 写编号边注

        如果边注需要在正文里留下一个明确的引用位置，可以使用 sidenote。它会在正文位置生成编号，并在右侧边注区域显示对应说明。

        效果类似这样：Tufte 的边注很适合写学习笔记中的补充推导编号边注适合承载“读到这里可能会问”的内容，例如公式变量说明、出处、反例或限制条件。，读者可以沿着正文主线阅读，也可以随时看右侧补充。

        写编号边注时要注意两点：

        • for 和 id 要一一对应，并且整篇文章里不要重复。
        • label、input、span.sidenote 最好紧挨着写，避免移动端折叠时位置错乱。

        边注里放图片

        边注不仅能放文字，也可以放图片。比如在推导激活函数、网络结构、几何图形时，可以把小图放到右侧，不打断正文。

        这种写法在我整理 Transformer FFN 推导时很有用：正文负责推导步骤，右侧放 ReLU、GELU、SwiGLU 的函数图像，读者不用来回跳转。

        主题还支持浅色/深色切换，以及代码块复制按钮。这些不需要在文章里额外写代码，正常使用 Markdown 代码块即可。

        我自己的排版习惯

        写 tufte-note 主题文章时，我会遵守几个简单规则：

        • 正文只放主线内容，避免被太多括号和补充说明切碎。
        • 术语解释、出处、变量说明放到边注。
        • 长代码不放边注，边注只放短提示。
        • 宽图要少用，只给真正需要横向空间的图。
        • 一篇文章里 HTML 片段不要太密集，否则 Markdown 的可维护性会下降。
