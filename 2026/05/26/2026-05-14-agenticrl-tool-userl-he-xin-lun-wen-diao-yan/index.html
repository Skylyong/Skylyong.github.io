        Agentic RL / Tool-Use RL 核心论文调研

        主题边界

        这里的主题不是传统 RL，而是围绕 LLM agent 的 post-training / reinforcement learning / tool-use / search / multi-turn credit assignment / multi-agent orchestration 展开。可以粗分为六条线：

        • RLHF / RLVR / preference optimization：如何用 reward、偏好、可验证答案训练语言模型。
        • reasoning scaling：CoT、self-consistency、process reward、DeepSeek-R1 这类推理能力来源。
        • tool-use / function calling / TIR：让模型知道什么时候调用工具、调用什么、如何把工具结果接回推理。
        • search agent / deep research agent：把检索、网页浏览、证据整合变成可训练的 agent 行为。
        • multi-turn agent RL：长程交互、延迟奖励、turn-level credit assignment。
        • multi-agent orchestration：多个 LLM/worker/工具之间的分工、通信拓扑和自然语言编排。

        引用量说明：引用数是 2026-05-14 附近从 Semantic Scholar、OpenAlex、Google Scholar 片段和论文页交叉看到的量级，变化很快，下面用“约”或“量级”表达。2025-2026 年论文引用还没充分积累，因此单独标成“新近核心”。

        最推荐阅读顺序

        0. 先补 post-training 和推理基础

        1. Training language models to follow instructions with human feedback
          InstructGPT / RLHF 的经典起点。先理解 SFT、reward model、PPO、KL penalty。
          https://arxiv.org/abs/2203.02155

        2. Direct Preference Optimization: Your Language Model is Secretly a Reward Model
          理解为什么后来很多 agent RL 工作会把 PPO、DPO、GRPO、RLOO 放在一起比较。
          https://arxiv.org/abs/2305.18290

        3. Chain-of-Thought Prompting Elicits Reasoning in Large Language Models
          agent 的“思考-行动”格式基本建立在 CoT 之上。
          https://arxiv.org/abs/2201.11903

        4. Self-Consistency Improves Chain of Thought Reasoning in Language Models
          读它是为了理解 test-time sampling / majority vote 为什么会成为后续 verifier、tree search、agent rollout 的基础。
          https://arxiv.org/abs/2203.11171

        5. Let’s Verify Step by Step
          process reward model 的代表作。后面 agentic RL 的难点，本质上就是把“哪一步有贡献”从数学题扩到工具调用和环境交互。
          https://arxiv.org/abs/2305.20050

        6. DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models
          GRPO 的重要来源之一。想做 RLVR / outcome reward / math reasoning，这篇绕不开。
          https://arxiv.org/abs/2402.03300

        7. DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning
          读它看 RL 如何诱导 long CoT、reflection、self-verification，以及冷启动数据 + 多阶段 RL 的工程范式。
          https://arxiv.org/abs/2501.12948

        1. 再读 agent / tool-use 的基础范式

        1. ReAct: Synergizing Reasoning and Acting in Language Models
          LLM agent 最核心范式之一：interleave reasoning and action。
          https://arxiv.org/abs/2210.03629

        2. MRKL Systems
          早期 neuro-symbolic / tool router 思路：LLM + 外部模块。
          https://arxiv.org/abs/2205.00445

        3. Toolformer
          自监督学习 API 调用：何时调用、调用参数、如何使用结果。
          https://arxiv.org/abs/2302.04761

        4. PAL: Program-aided Language Models
          把代码执行作为工具交给 Python runtime，是后续 code interpreter / TIR 的前身。
          https://arxiv.org/abs/2211.10435

        5. Gorilla / ToolLLM / API-Bank
          这三篇一起看：API grounding、真实 API 规模化、tool-use benchmark。
          Gorilla: https://arxiv.org/abs/2305.15334
          ToolLLM: https://arxiv.org/abs/2307.16789
          API-Bank: https://arxiv.org/abs/2304.08244

        2. 然后进入 search agent / deep research agent

        1. WebGPT
          较早把浏览器环境、引用收集、人类反馈结合起来的 web-browsing QA agent。虽然是 2021，但对 search agent 很关键。
          https://arxiv.org/abs/2112.09332

        2. Self-RAG
          学习“何时检索、如何批判检索结果、如何生成”的自反式 RAG。
          https://arxiv.org/abs/2310.11511

        3. Search-o1 / R1-Searcher / Search-R1 / ReSearch
          这组是 2025 search + reasoning + RL 的主线：从设计 workflow 到 outcome-based RL，再到 multi-turn retrieval reasoning。
          Search-o1: https://arxiv.org/abs/2501.05366
          R1-Searcher: https://arxiv.org/abs/2503.05592
          Search-R1: https://arxiv.org/abs/2503.09516
          ReSearch: https://arxiv.org/abs/2503.19470

        4. WebThinker
          从 QA search 扩展到 deep research：边搜索、边浏览、边写报告，并用 online DPO 强化 research tool 使用。
          https://arxiv.org/abs/2504.21776

        3. 重点读 tool-integrated reasoning / agentic RL

        1. ReTool
          动态交错自然语言推理和 code interpreter，并通过 outcome reward 学会何时用工具。
          https://arxiv.org/abs/2504.11536

        2. Tool-Star
          multi-tool collaborative reasoning 的重点论文：数据合成、难度课程、两阶段训练、hierarchical reward。
          https://arxiv.org/abs/2505.16410

        3. ET-Agent
          不只看答案对不对，还显式校准工具调用行为：冗余调用、不足调用、工具后过度思考等。
          https://arxiv.org/abs/2601.06860

        4. ArCHer
          比较早系统讨论 multi-turn LLM agent RL：长程、多轮、延迟奖励、层次化 actor-critic。
          https://arxiv.org/abs/2402.19446

        5. RAGEN
          multi-turn agent RL 的训练现象分析：Echo Trap、rollout shaping、reasoning-aware reward。
          https://arxiv.org/abs/2504.20073

        6. SWEET-RL / SPA-RL / ARPO
          三篇一起看 credit assignment：turn-level critic、stepwise progress attribution、multi-turn tool interaction 后的 policy optimization。
          SWEET-RL: https://arxiv.org/abs/2503.15478
          SPA-RL: https://arxiv.org/abs/2505.20732
          ARPO: https://arxiv.org/abs/2507.19849

        7. AgentGym-RL
          更偏系统和环境层：统一训练长程多轮交互 agent。
          https://arxiv.org/abs/2509.08755

        4. 最后看 multi-agent orchestration

        1. Generative Agents / CAMEL / AutoGen
          先看 multi-agent 社会模拟、role-playing 协作、工程框架化对话。
          Generative Agents: https://arxiv.org/abs/2304.03442
          CAMEL: https://arxiv.org/abs/2303.17760
          AutoGen: https://arxiv.org/abs/2308.08155

        2. Conductor
          直接对应“训练一个 conductor 用自然语言编排多个 worker LLM”：协作拓扑、任务分配、递归调用、test-time scaling。
          https://arxiv.org/abs/2512.04388

        新近核心论文

        这些论文大多在 2025-2026 年，引用还在积累，但和调研主题最贴近。

        论文 年份 方向 核心贡献 为什么要读
        Search-o1 2025 agentic search 把 search workflow 嵌进 reasoning model 作为 Search-R1 前置阅读
        R1-Searcher 2025 search RL outcome-based RL 激励模型主动搜索 看两阶段 search capability training
        Search-R1 2025 search + RL multi-turn search interactions + retrieved token masking 文档中 search-r1 的核心论文
        ReSearch 2025 search + RL 无监督 reasoning-step 数据，直接用 RL 学 search reasoning 和 Search-R1 对比读
        WebThinker 2025 deep research 自主搜索、网页导航、报告撰写 + online DPO 对 deep research agent 很关键
        ReTool 2025 tool-integrated reasoning 自然语言推理与 code interpreter 动态交错 文档中 retool 的核心论文
        Tool-Star 2025 multi-tool RL 多工具协同、数据合成、self-critic RL、层次 reward multi-TIR 重点
        ET-Agent 2026 tool behavior calibration 校准冗余/不足工具调用和工具后过度思考 从“能用工具”到“用得好”
        ArCHer 2024 multi-turn RL 层次化 multi-turn agent RL agentic RL 算法基础
        RAGEN 2025 multi-turn RL StarPO、Echo Trap、rollout shaping 理解训练稳定性和环境交互
        SWEET-RL 2025 multi-turn credit assignment turn-wise critic + collaborative reasoning tasks 对人机协作和多轮任务有帮助
        SPA-RL 2025 reward redistribution 把最终奖励分解成 stepwise progress 直接对 delayed reward 问题
        ARPO 2025 agentic RL algorithm 面向 multi-turn tool interaction 的 policy optimization 文档中 ARPO 的核心论文
        AgentGym-RL 2025 RL framework/env 统一长程多轮交互 agent 训练框架 想做系统复现可看
        Conductor 2025 multi-agent orchestration RL 训练 conductor 编排多个 worker LLM 对多智能体协作拓扑最相关
        rLLM 2025 framework post-training language agents 的开源 RL 框架 工程落地和 benchmark 很有用

        按研究问题的论文组合

        Q1：模型为什么会“会推理”？

        推荐顺序：

        1. CoT
        2. Self-Consistency
        3. Let’s Verify Step by Step
        4. DeepSeekMath
        5. DeepSeek-R1

        核心问题：从 prompt-induced reasoning 到 process reward，再到 outcome-based RLVR，推理能力到底来自数据、采样、reward 还是训练动态。

        Q2：模型为什么会“会用工具”？

        推荐顺序：

        1. MRKL
        2. Toolformer
        3. PAL
        4. API-Bank
        5. Gorilla
        6. ToolLLM
        7. ReTool
        8. Tool-Star
        9. ET-Agent

        核心问题：工具调用不是只有 function schema matching，还包括 tool selection、argument grounding、execution feedback、tool-result integration、overuse/underuse calibration。

        Q3：如何训练 search / deep research agent？

        推荐顺序：

        1. WebGPT
        2. Self-RAG
        3. Search-o1
        4. R1-Searcher
        5. Search-R1
        6. ReSearch
        7. WebThinker

        核心问题：从一次性检索到多轮搜索；从 RAG pipeline 到 agent search policy；从 answer reward 到 search trajectory reward。

        Q4：multi-turn agent RL 的难点是什么？

        推荐顺序：

        1. WebShop
        2. AgentBench
        3. ArCHer
        4. RAGEN
        5. SWEET-RL
        6. SPA-RL
        7. ARPO
        8. AgentGym-RL

        核心问题：长程交互里只有最终成败是不够的。关键是 reward redistribution、turn-level critic、环境反馈、trajectory filtering、探索稳定性。

        Q5：multi-agent orchestration 怎么读？

        推荐顺序：

        1. Generative Agents
        2. CAMEL
        3. AutoGen
        4. Conductor

        核心问题：从多 agent 模拟和 role-playing 协作，到工程框架，再到用 RL 训练 conductor 自动设计 worker 拓扑和自然语言指令。

        可能的研究切口

        • Agentic RL 算法：从 ArCHer、RAGEN、SWEET-RL、SPA-RL、ARPO 里找 credit assignment 和训练稳定性问题。
        • Search + RL：比较 R1-Searcher、Search-R1、ReSearch、WebThinker 的 search action 定义、reward、masking、generalization。
        • Tool-use 行为校准：沿着 Toolformer -> ToolLLM -> ReTool -> Tool-Star -> ET-Agent，看从“会调用”到“调用得少而准”。
        • Multi-agent conductor：从 AutoGen / CAMEL 的人工设计 topology，推进到 Conductor 的可学习 orchestration policy。
        • Agent benchmark 到训练闭环：用 WebShop、AgentBench、WebArena、BFCL、AgentGym-RL 找可验证 reward 和环境反馈。

        备注
