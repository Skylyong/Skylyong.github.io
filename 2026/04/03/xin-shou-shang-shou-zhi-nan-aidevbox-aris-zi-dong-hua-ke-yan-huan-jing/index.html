新手上手指南：AIDevBox + ARIS 自动化科研环境

本指南面向零基础新手，帮助你从零配置好 Claude Code / Codex 开发环境，并用 ARIS 实现”睡一觉、醒来论文写好了”的自动化科研工作流。

1. 整体架构概览

核心思路：

  • AIDevBox 提供隔离、开箱即用的 Docker 开发环境，内置 Claude Code、Codex、OpenCode 和 Python AI 工具链
  • cc-switch 在主机管理 API 密钥，配置实时同步进容器，无需重启
  • ARIS 是一套 Markdown 技能文件，驱动 Claude Code（执行）× Codex GPT-5.4（审稿）跨模型协作，实现从想法到论文的全自动流水线

8. 常见问题

Q：没有 OpenAI API，能用 ARIS 吗？
可以。使用 ModelScope 免费方案（Alt E），每天 2000 次调用，无需 Claude 或 OpenAI API。详见 docs/MODELSCOPE_GUIDE.md。

Q：没有 GPU，能跑实验吗？
可以。配置 gpu: vast，ARIS 会自动分析你的实验需求、搜索最便宜的 GPU、按需租用、实验完成后自动销毁。通常一次实验几美元。

Q：Codex MCP 配置好了，但审稿技能报错？
检查 ~/.codex/config.toml 中的模型是否为 gpt-5.4，然后运行 claude mcp list 确认 codex 条目存在。

Q：想在 Cursor / Trae / OpenClaw 里用 ARIS？
ARIS 支持多平台，见 docs/CURSOR_ADAPTATION.md、docs/TRAE_ARIS_RUNBOOK_EN.md、docs/OPENCLAW_ADAPTATION.md。

Q：过夜跑到一半中断了，进度丢了吗？
不会丢。ARIS 会保存 REVIEW_STATE.json，每轮结束后持久化状态。重新运行 /auto-review-loop 会自动从断点恢复。

Q：AIDevBox 里的 API Key 安全吗？
安全。Key 存在主机 ~/.claude/ 等目录，通过 Volume Mount 只读注入容器，不进入版本控制。

提示： ARIS 是方法论，不是平台。它的核心是纯 Markdown 技能文件，可运行在任何支持 Claude Code 风格 slash 命令的工具上。核心工作流永远是：Claude 执行 × 外部模型审稿 × 跨模型对抗协作。
