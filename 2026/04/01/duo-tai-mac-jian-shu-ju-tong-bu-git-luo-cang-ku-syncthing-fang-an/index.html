三、使用 Syncthing 实现两台 Mac 通过新加坡服务器同步文件

架构说明

三个节点都运行 Syncthing，新加坡服务器作为”始终在线”的中间节点，确保即使一台 Mac 离线，另一台上线后也能通过服务器同步到最新文件。

第一部分：新加坡服务器部署 Syncthing（Docker Compose）

1. 创建项目目录
2. 创建 docker-compose.yml
3. 启动服务
4. 配置服务器防火墙

如果使用云服务商（如 AWS、阿里云），也需要在安全组中放行以上端口。

5. 设置 Web UI 密码

首次访问 http://<服务器IP>:8384，Syncthing 会提示你设置用户名和密码。

务必立即设置！ 否则你的 Syncthing 管理界面暴露在公网上。

第二部分：两台 Mac 安装 Syncthing

方案 A：Homebrew 安装（推荐）

启动后自动打开 Web UI: http://localhost:8384

方案 B：下载官方 App

下载 Syncthing macOS app 或使用第三方图形界面 Syncthing macOS (GitHub)。

第三部分：三个节点互相连接

1. 获取每个节点的设备 ID

在每个节点的 Web UI 中：

操作 → 显示 ID （Actions → Show ID）

你会得到类似这样的设备 ID：

记录三个设备 ID：

  • Mac A 的设备 ID
  • Mac B 的设备 ID
  • 服务器的设备 ID
2. 互相添加设备

在 Mac A 的 Web UI 上：

  1. 点击 添加远程设备（Add Remote Device）
  2. 输入 服务器的设备 ID
  3. 设备名称填：SG-Server
  4. 地址填：tcp://<服务器IP>:22000（指定直连地址，更快建立连接）
  5. 保存

重复上述步骤，添加 Mac B 的设备 ID（地址可以留 dynamic）

在 Mac B 的 Web UI 上：

  1. 添加 服务器的设备 ID，地址填 tcp://<服务器IP>:22000
  2. 添加 Mac A 的设备 ID，地址留 dynamic

在服务器的 Web UI 上：

  1. 添加 Mac A 的设备 ID（地址留 dynamic）
  2. 添加 Mac B 的设备 ID（地址留 dynamic）

当一方添加另一方后，被添加方的 Web UI 通常会弹出确认请求，点击确认即可。

3. 创建共享文件夹
在 Mac A 上：
  1. 点击 添加文件夹（Add Folder）
  2. 配置：
    • 文件夹标签：Shared（自定义名称）
    • 文件夹 ID：shared-sync（三个设备必须一致！）
    • 文件夹路径：/Users/yourname/Sync（Mac A 上的实际路径）
  3. 切换到 共享 标签页：
    • 勾选 SG-Server
    • 勾选 Mac B
  4. 切换到 高级 标签页（可选）：
    • 文件夹类型：发送和接收（Send & Receive）
  5. 保存
在服务器和 Mac B 上：

当 Mac A 共享文件夹后，服务器和 Mac B 的 Web UI 会弹出”Mac A 想和你共享文件夹”的提示。

在服务器上 接受共享：

  • 文件夹路径设为：/var/syncthing/data/shared-sync
  • 共享对象勾选 Mac A 和 Mac B
  • 文件夹类型：发送和接收

在 Mac B 上 接受共享：

  • 文件夹路径设为：/Users/yourname/Sync
  • 共享对象勾选 Mac A 和 SG-Server
  • 文件夹类型：发送和接收

第四部分：网络拓扑优化 选择的方案

推荐的连接拓扑

三个节点互相连接，Syncthing 会自动选择最优路径。

在服务器上设置为 “Introducer”（可选）

在 Mac A 和 Mac B 的设备配置中，将服务器设为 Introducer（介绍人）：

编辑设备 → 勾选 Introducer

这样服务器上新增的共享设备/文件夹会自动同步到所有客户端。

注意事项

项目 建议
安全性 Web UI 8384 端口不要对公网开放，使用 SSH 隧道或设置强密码
磁盘空间 服务器磁盘需能容纳全部同步文件，注意监控
版本控制 Syncthing 支持文件版本控制，在文件夹设置→版本控制中开启
忽略文件 在同步目录下创建 .stignore 文件，写入不需要同步的文件模式
冲突处理 同时编辑同一文件会产生 .sync-conflict-* 冲突文件
NAT 穿透 三个节点都连服务器，不存在穿透问题；两台 Mac 直连可能需要中继
