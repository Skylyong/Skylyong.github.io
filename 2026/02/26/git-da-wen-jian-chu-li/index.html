        Git Large File Storage (LFS) 是 Git 的一个开源扩展，用于通过将大文件（如音频、视频、数据集、图形）替换为文本指针来管理这些文件，而将实际的文件内容存储在远程服务器上。

        2. 提交 .gitattributes 文件 (关键步骤)

        你必须将 .gitattributes 文件添加到 Git 仓库中，这样其他协作者才知道哪些文件是 LFS 文件。

        3. 正常添加和提交大文件

        现在，你可以像往常一样添加你的大文件了。Git LFS 会自动在后台处理它们。

        在执行 git push 时，你会看到 LFS 正在上传大文件的进度条，而不是普通的 Git 压缩对象进度。

          • 需要在 GitLab 配置文件（gitlab.rb）中开启 LFS 功能才能使用。

        总结流程

        1. 安装：git lfs install
        2. 指定：git lfs track "*.zip"
        3. 记录：git add .gitattributes
        4. 提交：git add file.zip -> git commit -> git push
